Enhance the AegisAI360 Endpoint Agent with three production-grade features:
1. Auto-update system
2. Telemetry collection
3. Remote command execution

The agent is written in Go and runs as a Windows service. Update the existing codebase to include the following:

====================================================
1. AUTO-UPDATE SYSTEM
====================================================
Implement a secure auto-update mechanism:
- The agent checks the backend every 5 minutes:
    GET https://aegisai360.com/api/agent/update?version=<currentVersion>
- If a newer version exists:
    - Download the update file (agent-update.exe)
    - Verify checksum (SHA256)
    - Stop the running service
    - Replace the current agent binary
    - Restart the service
- Log every update step to logs/update.log
- Fail safely if update fails (rollback to previous version)

====================================================
2. TELEMETRY COLLECTION
====================================================
Add a telemetry module that collects:
- CPU usage
- RAM usage
- OS version
- Hostname
- Running processes
- Active network connections
- Agent version
- Last update time

Send telemetry every 30 seconds to:
    POST https://aegisai360.com/api/agent/telemetry

Payload format:
{
  "agentId": "<uuid>",
  "hostname": "...",
  "os": "...",
  "cpu": "...",
  "ram": "...",
  "processes": [...],
  "connections": [...],
  "version": "...",
  "timestamp": "..."
}

====================================================
3. REMOTE COMMAND EXECUTION
====================================================
Add a command polling loop:
- Every 15 seconds, call:
    GET https://aegisai360.com/api/agent/commands/<agentId>

Supported commands:
- "ping" → respond with "pong"
- "get_info" → send full telemetry immediately
- "restart" → restart the agent service
- "update" → trigger auto-update immediately
- "exec:<command>" → run a safe shell command and return output

Send results to:
    POST https://aegisai360.com/api/agent/command-result

====================================================
4. FILE STRUCTURE
====================================================
Organize the code into these files:

/agent
  main.go
  service.go
  updater.go
  telemetry.go
  commands.go
  backend.go
  config.go
  utils.go

====================================================
5. SECURITY REQUIREMENTS
====================================================
- All backend communication must use HTTPS
- Validate SSL certificates
- Never log sensitive data
- Sanitize command execution
- Use UUID for agent identity
- Store config in:
    C:\Program Files\AegisAI360\Agent\config.json

====================================================
6. OUTPUT
====================================================
Provide:
- Complete Go code for all updated files
- Instructions to build the Windows .exe
- Example config.json
- Example backend responses
- Logging examples

Make sure the final result is fully functional and production-ready.
