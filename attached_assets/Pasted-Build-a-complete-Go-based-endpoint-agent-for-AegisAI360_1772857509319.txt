Build a complete Go-based endpoint agent for AegisAI360 that matches the backend API contract defined in the AegisAI360 system. The agent must run on Windows, Linux, and macOS, and must be compilable into a single binary (.exe for Windows). The agent must support registration, heartbeat, logs, command polling, command execution, and terminal command whitelisting.

==================================================
GO AGENT REQUIREMENTS
==================================================

1. REGISTRATION
---------------
On startup, the agent must:
- Read DEVICE_TOKEN from a constant in the code
- Collect:
    • hostname
    • OS (runtime.GOOS)
    • primary IP address
- POST to: /api/agent/register
- Save returned agent_id in memory

Payload:
{
  "device_token": "...",
  "hostname": "...",
  "os": "...",
  "ip": "..."
}

Response:
{
  "agent_id": "..."
}

2. HEARTBEAT LOOP
-----------------
Every 30 seconds:
POST /api/agent/heartbeat with:
{
  "agent_id": "...",
  "cpu_usage": <float>,
  "ram_usage": <float>,
  "status": "online",
  "timestamp": "<ISO8601>"
}

CPU and RAM can be stubbed for now (e.g., 10.0, 20.0).

3. LOG SENDER
-------------
Provide a function to send logs:
POST /api/agent/logs
{
  "agent_id": "...",
  "log_type": "info|error|system",
  "log_data": "..."
}

4. COMMAND POLLING
------------------
Every 5 seconds:
GET /api/agent/commands?agent_id=...

Response:
{
  "commands": [
    {
      "id": "...",
      "command": "ping | run_system_scan | terminal_exec",
      "params": { ... }
    }
  ]
}

5. COMMAND EXECUTION
--------------------

Supported commands:

A. ping
-------
Return:
{
  "status": "success",
  "output": "pong"
}

B. run_system_scan (stub)
-------------------------
Return:
{
  "status": "success",
  "output": "system scan executed (stub)"
}

C. terminal_exec
----------------
params:
{
  "command": "netstat -ano"
}

The agent must:
- Validate command against a whitelist
- Execute using exec.Command
- Capture stdout/stderr
- Return output

Whitelist:
- whoami
- ipconfig
- ifconfig
- ip a
- netstat -ano
- netstat -tunap
- ss -tunap
- tasklist
- ps aux
- dir <path>
- ls <path>

If command is not allowed:
Return:
{
  "status": "failed",
  "output": "command not allowed by whitelist"
}

6. COMMAND RESULT SENDER
------------------------
POST /api/agent/command-result
{
  "command_id": "...",
  "agent_id": "...",
  "status": "success|failed",
  "output": "..."
}

7. NETWORKING
-------------
- Use http.Client with timeout
- All requests must send JSON
- Handle non-200 responses gracefully

8. CROSS-PLATFORM
-----------------
- Must compile on Windows, Linux, macOS
- Must support building Windows .exe:
  GOOS=windows GOARCH=amd64 go build -o agent.exe main.go

9. CODE STRUCTURE
-----------------
- main.go
- helper functions:
    • registerAgent()
    • sendHeartbeat()
    • sendLog()
    • fetchCommands()
    • sendCommandResult()
    • handlePing()
    • handleRunSystemScan()
    • handleTerminalExec()
- Use goroutines for heartbeat loop

10. OUTPUT
----------
Produce:
- A complete main.go file
- Ready to compile
- No placeholders except DEVICE_TOKEN and backend URL
